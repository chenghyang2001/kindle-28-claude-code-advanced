<threat_model>
本 phase 為本機 in-memory CLI，無網路 / DB / 認證面。威脅面集中在函式契約的資料完整性。

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| 呼叫端 → list_tasks 回傳值 | 呼叫端持有回傳快照；若快照與內部 tasks 共用可變參照，外部改動會回滲內部狀態 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| ----------- | ---------- | ----------- | ------------- | ----------------- |
| T-01-01 | Tampering | list_tasks 回傳值別名 | mitigate | 以 `[replace(t) for t in tasks]` 做元素級複製；新增 test_list_tasks_element_mutation_does_not_leak 直接驗證 .done 不外洩（D-04） |
| T-01-02 | Denial of Service (輸入) | add_task 空/純空白名稱 | mitigate | 保留既有 `raise ValueError`，拒絕空名稱寫入，避免無意義髒資料累積 |
| T-01-03 | Supply chain | 相依套件 | accept | 本 plan 無任何新套件安裝；僅使用 stdlib `dataclasses`，pytest 為既有相依，無 legitimacy gate 需求 |
</threat_model>
