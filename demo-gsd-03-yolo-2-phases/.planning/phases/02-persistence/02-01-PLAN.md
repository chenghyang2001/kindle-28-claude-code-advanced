    - 中文不轉義：檔案文字含 "買牛奶" 字面（ensure_ascii=False 生效），非 \uXXXX。
    - Edge：save_tasks([], path) 寫出 "[]"，不報錯。

    在檔案頂部 import 區新增 `import json`（與既有 `import sys` 並列；標準庫，不新增依賴）。

    與既有純函式風格一致；不引入任何第三方套件。
  </action>
  <verify>

  <files>app.py</files>
  <behavior>
    - Happy：對 save_tasks 寫出的檔案 load_tasks(path) 回 list[dict{name,done}]，與存檔前相等。
    - Edge（缺檔）：load_tasks(不存在的路徑) 回 []（不拋例外）。
    - Error（壞 JSON）：對內容為 "{壞" 的檔案，load_tasks 讓 json.JSONDecodeError 往外拋（不被吞）。
  </behavior>
  <action>
    在 save_tasks 之後新增 `load_tasks(path)`（實作 PERS-02）。
    先處理缺檔：用 `os.path.exists(path)`（需 `import os`）或捕捉 `FileNotFoundError`，
    檔案不存在 → 回傳 `[]`（呼應 Phase 1「可預期的沒有回空」慣例，per 02-CONTEXT「load_tasks 行為」決策）。
    檔案存在則 `with open(path, "r", encoding="utf-8") as f:` 後 `return json.load(f)`。
    JSON 損壞時**不要** try/except 吞掉 → 讓 `json.JSONDecodeError` 自然往外拋（error case，per 02-CONTEXT 決策）。
    docstring 用繁體中文。實作者可自選缺檔判斷方式（exists 預檢 或 except FileNotFoundError），兩者皆符合決策。
  </action>
  <verify>
    <automated>PYTHONUTF8=1 python -c "import app,json,tempfile,os; d=tempfile.mkdtemp(); p=os.path.join(d,'t.json'); data=[{'name':'買牛奶','done':True},{'name':'寫程式','done':False}]; app.save_tasks(data,p); assert app.load_tasks(p)==data; assert app.load_tasks(os.path.join(d,'missing.json'))==[]; bad=os.path.join(d,'bad.json'); open(bad,'w',encoding='utf-8').write('{壞'); import json as J; ok=False;\nimport traceback\ntry:\n app.load_tasks(bad)\nexcept J.JSONDecodeError:\n ok=True\nassert ok; print('OK')"</automated>
  </verify>
  <done>load_tasks 存在；缺檔回 []、壞 JSON 讓 JSONDecodeError 外拋、正常檔還原 name 與 done。</done>
