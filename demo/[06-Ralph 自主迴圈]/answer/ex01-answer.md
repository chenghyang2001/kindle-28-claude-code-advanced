# 練習 01 解答 — 看懂 Ralph 自主迴圈

## 任務 1：迴圈狀態流

```
進入迴圈 (i=0)
   │
   ▼
① 產出  ── 讀 TASK.md → 做下一個未完成項 → 打勾
   │
   ▼
② 驗證  ── 跑測試 / 看 TASK.md 是否更新
   │
   ▼
③ 判斷收斂 ── 還有 [ ] 未完成項？
   ├─ 沒有了 ──────────────▶ ✅ 收斂出口（宣告完成）
   └─ 還有
        │
        ▼
   ④ 到上限 MAX？
     ├─ 還沒 ── i+=1 ──▶ 繞回 ①
     └─ 已達 ─────────▶ 🛑 backstop 出口（強制停止）
```

- **繞回點**：還有未完成項 **且** 未達上限。
- **跳出點**：① 收斂（正常）、② 撞上限（後盾）。

## 任務 2：終止條件 + 逃生條件

### 正常終止（2）
1. **收斂**：TASK.md 無 `[ ]` 未完成項 → 期望的結局。
2. **backstop 上限**：跑滿 MAX 輪未完成 → 強制停，防無限迴圈。

### 逃生煞車（出事立刻停，≥4）
1. **連續失敗**：同一步驟/測試連續 N 輪失敗 → 卡在解不開的問題，再跑只是燒錢，停下找人。
2. **零進展**：本輪產出與上輪完全相同、TASK.md 無變化 → 原地打轉，停。
3. **成本超標**：累積 token / 金額超過上限 → 立刻停（無人監督時最關鍵）。
4. **危險操作**：迴圈試圖做不可逆/高風險事（刪資料、改正式環境、push main）→ 攔下要人確認。

## 關鍵心得
- 正常出口靠「收斂 + 上限」；真正保命的是**逃生條件**。
- 設計原則：**寧可早停、報告卡在哪**，也不要讓無人監督的迴圈悶頭一直錯下去。

## 驗收

- [x] ≥2 停止條件（收斂 + backstop，另加 4 個逃生）
