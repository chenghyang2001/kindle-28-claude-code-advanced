# 練習 02 — 進階：為重複任務設計安全自主迴圈

## 情境說明
自主迴圈最危險的是「失控空轉」與「越改越糟」。設計時就要放護欄。

## 範例：護欄清單
```text
1. 最大迭代上限（硬上限，必達就停）
2. 每輪後跑測試/lint，紅燈就停並回報（不硬推）
3. 每輪 git commit 一個檢查點（可回滾）
4. 偵測「連續 N 輪無進展」即停（防空轉）
```

## 任務
### 任務 1
為一個你會重複很多次的任務，定義迴圈每輪的輸入/輸出。

### 任務 2
明列至少 3 個防止失控的護欄，並說明各自擋掉什麼失敗。

## 延伸思考
自主迴圈最危險的失敗模式是什麼？如何在設計時（而非事後）就堵住？
