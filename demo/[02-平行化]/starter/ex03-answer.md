# 練習 03 解答 — 把重複任務做成可重用 prompt 範本

## 選的重複任務
「讀一份 CSV → 算每個數值欄平均 → 輸出 Markdown 摘要」——每次拿到新資料都要做一次，是固化成範本的好對象。

## 交付物：slash command 範本

落檔位置：`.claude/commands/csv-summary.md`

```markdown
# /csv-summary — 產生 CSV 欄位平均摘要

讀取 $ARGUMENTS 指定的 CSV 檔，對所有數值欄位計算平均，
輸出為 Markdown 表到同目錄的 <檔名>-summary.md，並在終端印出處理列數與略過的缺值列數。

規則：
- 編碼 UTF-8、有表頭、逗號分隔
- 缺值（空字串）在計算平均時略過
- 平均四捨五入到 2 位

驗證：印出的列數可與 `wc -l` 對帳；數字可手算抽驗一欄。
```

## 怎麼換輸入重用
- `$ARGUMENTS` 是參數占位符：`/csv-summary ./data/sales.csv` 或 `/csv-summary ./data/2026q2.csv`
- 規則寫死在範本內 → 每次呼叫行為一致，不必重講脈絡

## 如何驗證輸出正確（範本內建）
1. **列數對帳**：範本要求印出處理列數，可與 `wc -l <csv>` 減表頭比對。
2. **抽驗一欄**：手算任一欄平均，對照輸出表。
3. **缺值處理**：故意放一列缺值，確認它被略過而非當 0。

## 心得
把「成功標準 + 驗證方式」一起寫進範本，重用時就不只省打字，還保證每次都可對帳——
這就是把練習 01、02 的單次經驗，沉澱成**可重複的資產**。
