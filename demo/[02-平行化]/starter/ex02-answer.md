# 練習 02 解答 — 設計一個 one-shot 的 CSV 分析 prompt

## 任務 1：完成此任務所需的「全部脈絡」清單

| 脈絡類別 | 本任務的值 |
|----------|-----------|
| 檔案路徑 | `./data/sales.csv` |
| 編碼 | UTF-8 |
| 格式 | 有表頭、逗號分隔 |
| 要算的欄 | `unit_price, quantity, total`（其餘欄忽略） |
| 缺值規則 | 空字串視為缺值，計算平均時略過 |
| 輸出位置/格式 | `./out/summary.md`，兩欄 Markdown 表（欄位 \| 平均值，四捨五入 2 位） |
| 自我驗證 | 同時印出「處理幾列 / 略過幾列缺值」 |

→ 把這 7 項備齊，prompt 才可能「一次到位」。少任一項，Claude 就得追問或猜。

## 任務 2：完整 one-shot prompt（已實際驗證一次成功）

```text
目標：分析 ./data/sales.csv，輸出每個數值欄位的平均值為 Markdown 表。

脈絡：
- 檔案：./data/sales.csv（UTF-8、有表頭、逗號分隔）
- 數值欄：unit_price, quantity, total（其餘欄位忽略）
- 缺值以空字串表示，計算平均時略過該值

輸出：
- 寫到 ./out/summary.md，兩欄表（欄位 | 平均值，平均四捨五入到 2 位）
- 同時在終端印出處理了幾列、其中幾列含缺值

成功標準：summary.md 能正常渲染；數字與手算一致。
```

### 實跑結果（一次到位、無追問）

```
| 欄位 | 平均值 |
|------|--------|
| unit_price | 112.5 |
| quantity | 3.33 |
| total | 320.0 |
```
- 處理 4 列、其中 1 列含缺值（B 列的 quantity/total 空）
- 對帳：unit_price=(100+150+80+120)/4=**112.5** ✅；quantity=(2+5+3)/3≈**3.33** ✅；total=(200+400+360)/3=**320.0** ✅

## 延伸思考：什麼時候 one-shot 反而比分步互動差？

- **需求本身還不清楚時**：你自己都還沒想好要什麼，硬寫 one-shot 只會把模糊放大。此時分步互動先探索、收斂需求更好。
- **高風險不可逆操作**（刪資料、改正式環境）：寧可分步、每步確認，也不要一次到位衝到底。
- **探索型任務**（不知道資料長怎樣）：先看一眼再決定怎麼處理，比一次寫死規則好。

> 結論：one-shot 適合「需求明確、可重複、低風險」的任務；探索與高風險場景反而該分步。
