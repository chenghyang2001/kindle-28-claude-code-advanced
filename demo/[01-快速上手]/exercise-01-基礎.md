# 練習 01 — 基礎：模糊指令 vs 結構化指令

## 情境說明
你要請 Claude 把一個資料夾內所有 `.txt` 檔合併成一個 `merged.txt`。同一任務，先用最隨意的一句話跑一次，再用結構化方式跑一次，量化兩者差距。

## 反例 / 正例對照（直接貼給 Claude 比較）
```text
# ❌ 模糊版（一句話）
把這些 txt 合併

# ✅ 結構化版（給脈絡 + 成功標準 + 輸出位置）
把 ./notes/ 底下所有 .txt（不含子目錄）依檔名排序合併成 ./out/merged.txt，
每個檔案內容前加一行「===== 檔名 =====」分隔，使用 UTF-8。
完成後印出合併了幾個檔、總行數。
```

## 任務
### 任務 1
用模糊版指令讓 Claude 執行，記錄：它問了你幾個回合？產出是否符合你心中預期？

### 任務 2
改用結構化版（成功標準 + 輸入範圍 + 輸出位置 + 編碼），再執行一次，記錄回合數與正確性。

### 任務 3
把兩次的「回合數 / 是否一次到位 / 需不需要返工」列成對照表，寫下結論。

## 驗收標準
- [ ] 兩個版本都實際跑過
- [ ] 有量化對照表（至少 3 個指標）
- [ ] 結論點出「結構化指令省下的具體成本」

## 完成後
將解答存入 `answer/ex01-answer.md`。
