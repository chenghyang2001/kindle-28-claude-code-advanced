# 練習 03 — 綜合挑戰：分批 migration 計畫

> 選配：完成前兩個練習後再挑戰

## 挑戰情境
為一個跨多檔的 migration（例如改一個被廣泛呼叫的 API 簽名）寫出分批執行計畫。

## 範例：擴展—收斂法（expand/contract）
```text
批次1：新增新簽名，舊簽名保留為 deprecated wrapper（不破壞既有呼叫）
批次2：逐檔把呼叫點改到新簽名（每檔一 commit，可驗證、可回滾）
批次3：所有呼叫點改完 -> 移除舊簽名
驗收：全測試綠 + grep 不到舊簽名殘留
```

## 限制條件
- 計畫需可逐批驗證與回滾
- 需有「全部完成」的客觀驗收條件（例如 grep 不到舊用法）
