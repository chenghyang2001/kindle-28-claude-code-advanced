# 練習 01 — 基礎：設計三 agent 團隊

## 情境說明
多 agent 各有乾淨 context、各司其職，由主 agent 當架構師整合，常勝過單 agent 一條龍。

## 範例：分工與交付
```text
研究 agent：只讀，產出「相依地圖 + 風險清單」
實作 agent：依研究結論在 src/ 實作，產出「程式碼 + 單元測試」
審查 agent：只讀，產出「必改/建議清單」
主 agent ：派工、整合、處理衝突；不下場寫業務碼
```

## 任務
### 任務 1
為一個任務畫出三 agent 的分工與交付物（誰產出什麼給誰）。

### 任務 2
定義 agent 之間的「合約」邊界（誰能寫哪些檔、誰只能讀）。

## 驗收標準
- [ ] 三 agent 職責不重疊
- [ ] 有明確的讀/寫邊界（合約）
