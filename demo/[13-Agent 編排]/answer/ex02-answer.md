# 練習 02 解答 — 用 agent debate 過濾爭議決策

## 任務 1：對抗審查的不同角度（以「採用方案 X」為例）

| 角度 | 提問 | 抓的問題類型 |
|------|------|-------------|
| 正確性 | 邏輯/邊界有沒有漏？ | 邏輯錯誤、漏掉的 case |
| 安全性 | 有沒有引入漏洞/可被攻擊？ | 安全風險 |
| 可維護性 | 3 年後接手的人會不會罵？ | 技術債、可讀性 |

## 為什麼要不同角度（非三個一樣）
- 三個一樣 = 三雙眼看同方向，抓一樣問題、漏一樣盲點，本質跟一個沒兩樣。
- 不同角度涵蓋面廣：方案可能「邏輯對但有漏洞」「又對又安全但難維護」，多角度才照到不同失敗模式。
- 比喻：一樣 = 同一盞燈打三倍亮；不同 = 三方向各打一盞，才照到死角。

## 任務 2：合議裁決（範例）
- 正確性：通過（邏輯無漏）
- 安全性：反對（缺輸入驗證）
- 可維護性：反對（耦合過高）
- 結果：>= 2 反對 → **打回**。

## 為什麼多數決（防什麼）
- 防「單一審查者也會誤判」。全信一個 → 它一錯整個判斷跟著錯。
- 多數決用群體智慧稀釋個體誤差；「兩個反對才打回」= 問題要明顯到多數看得出才算，過濾單人雜訊、也在某審查者失常時靠其餘把關。
- 深層：對抗驗證不是找完美審查者，而是承認每個都不完美、用結構彌補（多角度補涵蓋、多數決補誤差）。

## 延伸思考：多 vs 單 agent
- 多 agent 優於單：任務夠大可拆、或正確性重要值得多角度驗；小任務用多 agent = 殺雞用牛刀。
- 協調成本藏在：① context 同步 ② 整合拼湊/解矛盾 ③ 溝通延遲與打回票重工。多 agent＝用協調成本換品質與規模。

## 關鍵收穫
- 對抗驗證靠「多角度 + 多數決」過濾「看似對其實錯」。
- 多 agent 不免費，協調成本在 context 同步與整合。
