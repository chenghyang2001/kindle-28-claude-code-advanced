## 卡關與解法
- 卡關：Reviewer 改了 src/ 為什麼要當警報？
- 根本原因：只想到「違反邊界」，沒看到更致命的中立性破壞。
- 解法：Reviewer 一改 src 就從審查者變作者，再審會偏袒自己 → 掉回自我審查盲點，等於失去當初請它的唯一理由。

## 關鍵收穫（3 句以內）
- 拆多 agent 的核心是乾淨 context + 獨立審查；指揮者不下場、邊界講死才不亂。
- 對抗驗證靠多角度（補涵蓋）+ 多數決（補誤差）過濾「看似對其實錯」。
- 大任務編排靠合約（Never modifies）+ 信號檔接力 + 越權偵測，用結構管理不完美個體。
