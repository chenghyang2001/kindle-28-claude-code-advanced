# 練習 02 — 進階：用 agent debate 過濾爭議決策

## 情境說明
對抗式驗證：派多個審查者用「不同角度」挑同一個結論，靠合議過濾掉「看似對其實錯」。

## 範例：多角度審查
```text
對「採用方案 X」這個決策，派 3 個審查者各持一角度：
- 正確性：邏輯/邊界有沒有漏
- 安全性：有沒有引入風險
- 可維護性：3 年後接手的人會不會罵
裁決：>= 2 個審查者反對則打回，否則通過。
```

## 任務
### 任務 1
為一個有爭議的決策，設計 2-3 個對抗審查者的提問角度。

### 任務 2
用多數決或合議產出結論，記錄每個審查者的判斷。

## 延伸思考
多 agent 何時優於單 agent？協調成本（context 同步、整合）藏在哪？
