# 練習 01 — 基礎：先計畫再動手（Plan Mode）

## 情境說明
對中等任務，先讓 Claude 產出計畫、由你審查，再決定執行，能大幅降低「做歪重來」。

## 範例：觸發與審查
```text
# 進入 Plan Mode（Claude Code 會先規劃、不動手）後給任務：
為這個 CLI 加一個 `export` 子指令，把目前設定輸出成 JSON。
先給我實作計畫（要改哪些檔、新增哪些函式、測試怎麼寫），等我說 ok 再寫。
```
審查重點：步驟是否最小可獨立、有沒有漏掉測試、會不會動到不該動的檔。

## 任務
### 任務 1
寫一個會觸發 Plan Mode 的任務描述（明確要求「先計畫、等確認」）。

### 任務 2
檢視 Claude 的計畫，標出至少 1 個你會修改的步驟，並說明為什麼。

## 驗收標準
- [ ] 拿到一份分步計畫（非直接動手）
- [ ] 對計畫做了實質審查（不是直接 ok）

## 完成後
將解答存入 `answer/ex01-answer.md`。
