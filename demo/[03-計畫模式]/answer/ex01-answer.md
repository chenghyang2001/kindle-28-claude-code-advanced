# 練習 01 解答 — 計畫模式（Plan Mode）

> 以互動問答方式完成（2026-06-20）。逐題推進，使用者答不出時由 Claude 解析。

## Q1：Plan Mode 是什麼？為什麼要它？

**重點：** Plan Mode 是讓 Claude「**先規劃、不動手**」的模式——它會先把要改哪些檔、分幾步、怎麼測攤出來給你看，**等你核可才開始寫**。

**白話：** 像動手術前先看 X 光片、討論手術計畫，而不是病人一躺上去就開刀。你在「還沒寫半行 code」的階段就能攔住方向錯誤，成本最低。

## Q2：怎麼審查一份計畫？

三個檢查點：

1. **看範圍有沒有膨脹**：應該只動該動的檔案，不要自作主張順手把整個模組重構掉。
2. **看步驟是不是最小、可獨立**：每一步都能單獨做、單獨驗證，出錯才好定位。
3. **看有沒有測試這一步**：沒有測試的程式碼，只是「我覺得會動」而已。

**心法：** 你不是在看它寫得多努力，而是問三件事——範圍有沒有爆、步驟夠不夠細、怎麼證明做對了。任一項不過關，就**把計畫打回去重出**，而不是回頭改成品。

## Q3：哪種任務值得先進 Plan Mode，哪種直接做就好？

- **值得**（先計畫）：改動大、碰很多檔案、做錯難收回。例：**重構整個登入流程**（動 auth 中介層、token 驗證、DB 欄位）。
- **不值得**（直接做）：範圍極小、一眼看完、做錯好回頭。例：**改錯字、調設定值、補一行註解**。

**判斷依據三個字：範圍、風險、可逆。** 範圍越大、做錯代價越高、越難 undo → 越該先看計畫。

**一句話心法：** 任務越像「心臟手術」就越要先進 Plan Mode；越像「貼 OK 繃」就別浪費那個力氣。
