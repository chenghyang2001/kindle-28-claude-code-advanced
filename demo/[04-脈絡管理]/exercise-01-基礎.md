# 練習 01 — 基礎：辨識 context rot（脈絡腐化）

## 情境說明
session 太長時，Claude 會開始忘記早期決定、重複問已答過的事、或拿舊版檔案內容作答。學會辨識徵兆並選對處置。

## 範例：處置指令
```text
/clear      # 清空脈絡，完全重來（任務切換、已偏離主題時）
/compact    # 壓縮脈絡，保留重點繼續（同一任務但對話過長時）
/compact 保留：目前在做的步驟、關鍵檔案路徑、已確定的設計決策
```

## 任務
### 任務 1
刻意把一個 session 撐長（連續多個子任務），記錄 3 個「脈絡腐化」的具體症狀。

### 任務 2
針對你的情況，判斷該 `/clear`、`/compact` 還是繼續，並說明判斷依據。

## 驗收標準
- [ ] 至少記錄 3 個可辨識的腐化症狀
- [ ] 對 clear/compact/繼續 做出有依據的選擇

## 完成後
將解答存入 `answer/ex01-answer.md`。
