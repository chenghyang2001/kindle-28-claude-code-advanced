# 練習 02 — 進階：設計壓縮必留清單

## 情境說明
`/compact` 會丟掉細節。先定義「壓縮時一定要保留什麼」，避免壓縮後關鍵 ID/路徑遺失。

## 範例：compact instructions（可放 CLAUDE.md 或 settings）
```markdown
## 壓縮時必留
1. 進行中任務的當前步驟 + 下一步
2. 關鍵 ID（PR 編號、task id、notebook id…）
3. 已修改的檔案絕對路徑（含已 staged 未 commit）
4. 本 session 的重要決策與使用者明確偏好
5. 等待中的背景任務（排程時間、預期完成）
```

## 任務
### 任務 1
列出你的工作裡至少 5 類「壓縮後一定要保留」的資訊。

### 任務 2
寫一段「壓縮後接續摘要」範本（任務 / 進度 / 關鍵 ID / 待辦四段）。
