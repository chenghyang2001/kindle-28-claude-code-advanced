# 練習 02 解答 — 設計壓縮必留清單

## 任務 1：壓縮後一定要保留的 5+ 類資訊

| # | 必留類別 | 忘了會怎樣 |
|---|---------|-----------|
| 1 | 當前步驟 + 下一步 | 不知道做到哪、接下來要幹嘛 |
| 2 | 關鍵 ID（PR / task / notebook / 背景任務 id） | 推不回來，忘了只能重查或永久遺失（最致命） |
| 3 | 已改檔案絕對路徑（含已 staged 未 commit） | 動錯檔、重複改同一處 |
| 4 | 重要決策 + 使用者明確偏好（例：影片依序觸發、間隔 30 秒） | 違反使用者要求 |
| 5 | 等待中的背景任務（排程時間 / 預期完成） | 不知道有東西在跑 → 重複觸發或漏接結果 |
| 6 | 失敗紀錄與已排除的做法 | 重蹈覆轍、把試過不行的再試一遍 |

**判斷原則**：忘了就「做錯／推不回來」→ 必留；能從程式碼或現場重查到 → 可壓掉。

## 任務 2：「壓縮後接續摘要」範本

```markdown
## 本 Session 目前狀態（壓縮後接續）

**任務**：<一句話講清楚現在在做什麼>

**進度**：
- 已完成：<...>
- 進行中：<卡在哪一步>
- 下一步：<接下來要做什麼>

**關鍵 ID／路徑**：
- notebook_id: <...>
- task id: <...>
- 已改檔案（絕對路徑，含已 staged 未 commit）: <...>

**待辦**：
- [ ] <剩下要做的事>
- [ ] 背景任務 <id>，預定 <時間> 完成
```

### 填好範例

```markdown
## 本 Session 目前狀態（壓縮後接續）

**任務**：第四課脈絡管理，做進階練習（壓縮必留清單）。

**進度**：
- 已完成：基礎題（ex01）、進階任務 1（必留 6 類）
- 進行中：進階任務 2 寫範本
- 下一步：延伸思考 + 填 STEP_LOG

**關鍵 ID／路徑**：
- 改過檔案：skills/say_ui.py、skills/dist/say_ui.exe、CLAUDE.md
- 最新 commit：b5e018a

**待辦**：
- [ ] 完成 ex02-answer.md
- [ ] 回答延伸思考（為何長 context 反而降品質）
- [ ] 決定做綜合題還是跳第五課
```

## 延伸思考：為什麼更長的 context 常常降低品質？

- **稀釋（needle in haystack）**：關鍵指令被海量無關內容淹沒，模型注意力被稀釋，重要約定權重下降。
- **早期內容被擠出**：白板滿了，最早講的決策最先被擦掉 → 自相矛盾。
- **舊雜訊干擾**：過時的嘗試、已棄方案還留在脈絡裡，模型可能照舊資訊作答。
- **結論**：context 不是越多越好；**乾淨、聚焦、關鍵資訊落檔** 才是高品質的關鍵（context hygiene）。
