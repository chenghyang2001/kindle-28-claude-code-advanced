# 練習 01 解答 — 辨識 context rot（脈絡腐化）

## 任務 1：3 個以上腐化症狀

1. **重複問已回答過的事**：早就講過「用 PostgreSQL」，它又問「你想用哪個資料庫？」。
2. **拿舊版內容作答**：檔案中途改過，它仍引用改之前的內容。
3. **忘記已確定的決策、自相矛盾**：說好「小寫加連字號」卻又產出 `MyFile.py`；前後回答方案互相打架且講不出理由。
4. **答案變空泛**：不再貼著本專案的具體脈絡（檔名/變數/先前約定），開始講通則。
5. **關鍵 ID／路徑「失憶」**：要你把 notebook_id、task ID、絕對路徑再貼一次。

> 比喻：Claude 的記憶像一張有限大小的白板，對話越長越滿；滿了要寫新的就得擦掉最舊的——所以腐化幾乎都「從最早講的東西開始忘」。

## 任務 2：/clear、/compact、繼續 怎麼選

**判斷依據（一句話）**：接下來要做的事，跟前面的脈絡還有沒有關係。

| 處置 | 時機 | 例子 |
|------|------|------|
| `/clear` | 接下來任務跟前面**無關**，或脈絡已髒到不如重來 | 語音工具做完，要去上第五課 |
| `/compact` | **同一任務**但對話太長，前面有「必須留」的決策/ID/路徑，但雜訊也多 | 長 pipeline 做到一半；壓縮時指定保留清單 |
| 繼續不動 | 還沒出現腐化，或再幾步就做完（壓縮反而可能壓掉還需要的細節） | 沒壞就不要修 |

**最易踩的坑**：關鍵 ID 還沒落檔前，**不要先 `/clear`**（清掉永久找不回）。正確順序＝先把關鍵 ID／路徑寫進檔案，再壓縮或清空。

- [x] 對 clear/compact/繼續 做出有依據的選擇
