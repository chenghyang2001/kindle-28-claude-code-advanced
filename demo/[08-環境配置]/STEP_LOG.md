2. 基礎：盤點本機 `~/.claude` 真實環境，列 5 個可優化點，並**實際修掉 2 個真問題**：
   - #1 安全：全域 settings `env` 明文 `GOOGLE_DOCS_CLIENT_ID/SECRET` → `setx` 成 OS 環境變數後從 settings 刪除。
   - #2 Bug：3 條 env 路徑被存成字面 JSON 跳脫 ``/`\t` → 解析出控制字元 → 認證檔找不到（正是 AUTH CHECK 三個 MISSING 的元兇之一）。
   - 解答存 `answer/ex01-answer.md`。

## 卡關與解法
- 卡關：修 #2 時 byte 取代、原始文字取代都「0 處」，改不動。
  - 根本原因：檔案裡存的是**字面 JSON 跳脫序列**（``、`\t`），不是真的控制字元 byte，所以照控制字元或照表面字串都比對不到。
  - 解法：別跟跳脫層硬幹，繞到語意層——用 `json.loads` 載入，再用 `chr(92)`（反斜線）重新賦值三個路徑、`pop` 掉兩個密鑰鍵，`json.dumps` 寫回。一次成功，且驗證 JSON 合法 + 結構（permissions/hooks/model）完整無副作用。

## 關鍵收穫（3 句以內）
- settings 三層：個人 local > 專案 > 全域；金鑰永遠走環境變數/.env 不入 settings。
- 改設定檔前先備份、改後驗證「JSON 合法 + 頂層鍵/權限/hook 數量不變」才算安全。
- 遇跳脫序列問題不要在 byte 層硬比對，繞到 json 語意層用 `chr(92)` 賦值最穩。
