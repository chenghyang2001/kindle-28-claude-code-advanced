# 練習 01 — 基礎：寫一份精煉 CLAUDE.md

## 情境說明
CLAUDE.md 是「給 Claude 看的專案簡報」。好的 CLAUDE.md 讓 Claude 一開場就懂你的專案。

## 範例：5 段式骨架
```markdown
# <專案名>
## 專案概述      <!-- 一段話：這是什麼、解決什麼 -->
## 架構          <!-- 目錄結構 / 系統圖 -->
## 技術堆疊      <!-- 框架 + 版本 -->
## 執行方式      <!-- build / test / run 指令 -->
## 重要架構決策  <!-- 為什麼選 A 不選 B -->
```

## 任務
### 任務 1
為一個現有專案寫出上面 5 段骨架。

### 任務 2
填入該專案「真實的」執行指令與架構（不要寫通用廢話）。

## 驗收標準
- [ ] 5 段齊全
- [ ] 執行指令可直接複製跑
- [ ] 至少 1 條「為什麼這樣設計」的決策

## 完成後
將解答存入 `answer/ex01-answer.md`。
