# 練習 02 — 進階：把 review → simplify 串成固定流程

## 情境說明
單一外掛價值有限，串起來才是工作流。例如改完先 review 抓問題、再 simplify 去冗餘。

## 範例：串接順序
```text
1) /code-review   -> 收集必改項，先修掉 bug/風險
2) /simplify      -> 對已正確的程式碼去冗餘、提可讀性
3) 人工複檢 diff  -> 確認 simplify 沒改變行為
```

## 任務
### 任務 1
定義你的 review → simplify 串接順序與每步「通過才進下一步」的條件。

### 任務 2
在一個真實 diff 上跑完整條流程，記錄每步攔下/改善了什麼。

## 延伸思考
外掛何時幫倒忙？如何避免外掛之間職責重疊（例如兩個都想改同一段）？
