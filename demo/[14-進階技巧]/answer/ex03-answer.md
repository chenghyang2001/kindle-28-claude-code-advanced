# 練習 03 解答 — 我的進階技巧手冊（5 個 pattern）

### Pattern 1：多步拆解 + 成功標準
- 觸發條件：想做複雜（3+ 元件）的系統。
- 做法：先拆步驟 + 每步驗收標準，確認後逐步實作，不 one-shot。
- 實例：第 14 課登入系統（「做一個登入系統」→ 4 步 + 「錯誤密碼 401、測試全綠」）。

### Pattern 2：先張網再跳
- 觸發條件：重構或修改沒有測試的舊程式碼。
- 做法：動手前先補 characterization test 釘住現況、測試全綠，才小步改、每綠燈 commit。
- 實例：第 11 課安全重構、第 10 課 legacy 補測試。

### Pattern 3：二分搜尋定位
- 觸發條件：一大堆改動或一長段程式碼裡找問題在哪。
- 做法：對半砍、每次排除一半（git bisect 或手動對半註解）。
- 實例：第 14 課 100 個 commit 7 次抓到兇手。

### Pattern 4：對抗驗證
- 觸發條件：重要又有爭議的決策，或一定要正確的產出。
- 做法：派多個審查者各站不同角度挑，多數決裁定。
- 實例：第 13 課 agent debate（正確性/安全性/可維護性，2 票反對打回）。

### Pattern 5：指揮者不下場
- 觸發條件：大任務拆給多個 agent 平行做。
- 做法：主 agent 只派工/整合/處理衝突，不寫業務碼；合約定邊界、信號檔接力。
- 實例：第 13 課完整編排、日常 writer+QA+reviewer 鐵律。

## 共同精神
把大的/模糊的/危險的東西，拆成小的/明確的/可驗證的步驟，並在每步保留「能判斷對錯、能回頭」的能力。

## 關鍵收穫
- 每個 pattern 的價值一半在「做法」、一半在「觸發條件」（知道何時用）。
- 五招同源：隔離變因、明確驗收、保留回頭路。
