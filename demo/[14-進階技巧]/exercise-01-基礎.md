# 練習 01 — 基礎：用多步拆解重寫弱 prompt

## 情境說明
弱 prompt＝一句話要複雜系統。改成「先拆步 + 給成功標準」能大幅提升一次到位率。

## 範例：弱 vs 強
```text
# 弱
做一個使用者登入系統

# 強（多步拆解 + 成功標準）
分 4 步做登入：1) 資料表 schema 2) 註冊/登入 API（含錯誤碼）
3) 密碼 bcrypt + JWT 4) 端對端測試（註冊->登入->存取受保護資源）。
先列步驟與每步驗收標準，我確認後再逐步實作。
成功標準：錯誤密碼回 401；token 過期回 401；測試全綠。
```

## 任務
### 任務 1
找一個你寫過的弱 prompt，標出它的 3 個弱點。

### 任務 2
改寫成多步拆解 + 成功標準版，實跑驗證效果差異。

## 驗收標準
- [ ] 明確指出原版 3 個弱點
- [ ] 改寫版有可驗收的成功標準
