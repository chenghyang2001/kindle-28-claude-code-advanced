# 練習 01 — 基礎：30 分鐘 onboard 陌生 codebase

## 情境說明
進到不熟的 repo，先建立高階地圖再深入，比一頭栽進細節省時得多。

## 範例：onboarding 指令流
```text
/init                         # 產一份 CLAUDE.md 草稿（架構摘要）
# 或直接請 Claude：
「不要改任何檔。先給我這個 repo 的高階架構：主要模組、進入點、資料流，
 用一張文字圖呈現，並列出 3 個我該先讀的關鍵檔。」
```
```bash
git log --oneline -20         # 近期演化
git ls-files | sed 's@/[^/]*$@@' | sort -u | head   # 目錄概覽
```

## 任務
### 任務 1
挑一個你不熟的開源 repo，請 Claude 產出模組地圖與資料流（限它只讀不改）。

### 任務 2
標出 3 個你還看不懂的地方，逐一追問到懂。

## 驗收標準
- [ ] 有一張高階架構/資料流圖
- [ ] 至少 3 個疑點被追問釐清

## 完成後
將解答存入 `answer/ex01-answer.md`。
