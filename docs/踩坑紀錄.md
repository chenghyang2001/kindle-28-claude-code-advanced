## [課程03] 計畫模式（2026-06-20）— 衍生的 Python 工具鏈踩坑

做答案的「文字轉語音」工具（`say.py` / `say_ui.py`）+ 打包 exe + 上 GitHub 時踩到的坑：

### 1. PowerShell 子行程不繼承 Git Bash 的 CWD

- 踩坑：用 PowerShell 播 `q2-answer.mp3`（相對路徑），卻在 `C:\Windows\System32` 找不到檔。
- 根本原因：從 Git Bash 呼叫 `powershell` 時，子行程工作目錄不是 bash 的 CWD。
- 解法：先把檔案 `shutil.copyfile` 到 ASCII-only 暫存路徑（`tempfile`），再用絕對 ASCII 路徑餵 PowerShell。

### 2. 中文／方括號路徑傳進 PowerShell 會編碼壞掉

- 踩坑：路徑含 `[03-計畫模式]` 時 PowerShell 開檔失敗。
- 解法：同上——一律走 ASCII 暫存路徑。**跨 Git Bash ↔ PowerShell 的鐵則：別傳中文/方括號路徑。**

### 3. Python 不認 `/c/...`（MSYS 風格）路徑

- 踩坑：`python say.py -f "/c/Users/.../q3.txt"` → 找不到檔。
- 根本原因：MSYS 的 `/c/` 前綴只有 Git Bash 認得，Python 在 Windows 會當成 `C:\c\...`。
- 解法：傳給 Python 用 `C:/Users/...`（正斜線）開頭，Python 吃得下。

### 4. PowerShell `Register-ObjectEvent` 沒清理 → exit 2 偽陰性

- 踩坑：MP3 明明播完了，Python 卻判「播放失敗」。
- 根本原因：MediaFailed 事件訂閱者與背景 job 未清除，runspace 拆除時 PowerShell exit 2；而 Python「只看 returncode != 0」就誤判。
- 解法：腳本結尾 `Get-EventSubscriber | Unregister-Event; Get-Job | Remove-Job -Force` + 明確 `exit 0`。

### 5. exe 圖示 ≠ 視窗標題列圖示

- 踩坑：`pyinstaller --icon` 換了 exe 圖示，tkinter 視窗標題列卻還是 Tk 羽毛。
- 根本原因：`--icon` 只改 exe 檔案本身的圖示（工作列/檔案總管）；標題列那顆是 tkinter 自己的 window icon，必須程式碼 `root.iconbitmap()`。
- 解法：加 `resource_path()`（用 `sys._MEIPASS` 相容 onefile）+ `root.iconbitmap()`；打包加 `--add-data "app_icon.ico;."` 把 ico 包進去。

### 6. 用乾淨 venv 打包，exe 大幅變小（46MB → 15MB）

- 踩坑：全域 Python 打包出 46MB exe（夾帶 numpy.libs 等無關 DLL）。
- 解法：建 venv 只裝必要 4 套件（edge-tts / pywin32 / pyinstaller / pillow）再打包 → 15MB。**打包前先進乾淨 venv，可重現又不夾垃圾。**

### 7. Git：內嵌 repo 陷阱 + gitignore 方括號要跳脫

- 踩坑：(a) 第二課 worktree 練習在 `sandbox/` 留下 3 個內嵌 git repo，`git add` 會當成壞掉的 gitlink；(b) 想用 `demo/[02-平行化]/sandbox/` 擋卻無效。
- 根本原因：(a) 內嵌 repo 要 `git rm -rf --cached`；(b) gitignore 中 `[ ]` 是字元類別，`[02-平行化]` 不會匹配字面目錄名。
- 解法：(a) `git rm -rf --cached` 移除；(b) 跳脫成 `demo/\[02-平行化\]/sandbox/`。順手把 64MB 版權書 `pdf/` 也擋掉（公開 repo 不放）。
