## 核心主旨

外掛程式 (Plugins) 是 Anthropic 官方的擴充機制，能把 Claude Code 從「單純對話工具」升級成「可高度客製化的自動化開發引擎」。

掌握本章 = 從**被動的 AI 使用者** 蛻變成 **主動的 AI 架構師**。

---

## 一、外掛的結構：plugin.json

外掛的神經中樞是一份 `plugin.json` 清單檔（比喻成「系統化廚房的設計圖／食譜」），不是黑箱。它可組合以下**五大元素**：

| 元素 | 性質 | 說明 |
|------|------|------|
| **Skills** | AI 自主 | 模型依當下上下文自動判斷並呼叫（例：偵錯時自動抓取最新系統日誌） |
| **Commands** | 人類主導 | 使用者明確輸入的斜線指令（如 `/deploy`、`/run-test`），控制權在人手上 |
| **Agents** | 專職專家 | 預先封裝好身份與規範的專門 AI（如「資安專家」），隨時召喚，免重貼提示詞 |
| **Hooks** | 事件驅動 | 特定事件自動觸發（如存檔時自動喚醒資安 agent 檢查 SQL injection） |
| **MCP servers** | 對外橋樑 | 連接外部世界（發警告到 Slack、自動建 Jira 任務） |

五者組合 → 不再是輔助腳本，而是「具備完整感知與執行能力的自動化解決方案」。

## 二、官方生態系（規模化）

- 自 **2025/10** 起，Anthropic 維護一個高品質官方外掛庫，把頂尖開發者經驗變成全球可共享的標準資產。
- 範例：
  - **commit commands** — 把繁瑣 git 流程變一鍵捷徑。
  - **PR Review Toolkit** — 內含 **6 個協作代理人**：
    1. 掃語法 →
    2. 找安全漏洞 →
    3. 評效能 →
    4. 查單元測試 →
    5. 對比架構指南 →
    6. 彙整成審查報告（像專案經理）
- 安裝極簡：`plugin install <名稱>`；輸入 `plugin` 會開啟 Discover 探索分頁（含官方市集 + 社群市集）。

## 三、🔴 最嚴肅的部分：安裝前三大安全守則

社群市集可一行指令載入「不認識的程式碼」，等同「引狼入室」。作者強調必修三守則：

1. **審查原始碼 (Review the source code)**
   官方外掛程式碼公開在 GitHub。重點不是逐行 code review，而是**抓異常行為**
   （例：一個只該格式化文字的外掛，卻有把本地環境變數偷傳外部伺服器的網路請求）。

2. **驗證作者 (Verify the author)**
   認明正字標記。官方唯一正確儲存庫路徑：`anthropics/claude-plugins-official`
   （書中 2026/05 更新時的驗證狀態）。
   社群外掛不一定惡意，但**沒經官方強制審查** = 風險自負。

3. **理解權限 (Understand the permissions)** — 最關鍵
   `plugin.json` 的 `allowedTools` 屬性直接擴張 Claude 的動作範圍，等於發「最高安全通行證」。
   盲裝含系統層權限的外掛 = 給 AI「上帝模式 (God Mode)」。

### 為什麼危險

不需到「AI 背叛人類」那麼科幻，光是**幻覺 (hallucination)** 就夠——
若模型誤判要「清理專案」，加上刪除權限 + 自動執行的 Hook，
可能一瞬間複寫掉你一個月的程式、改掉伺服器核心設定。

> 清楚知道「交出了多少控制權」，是區分**盲目跟風者**與**專業架構師**的決定性差異。

## 四、結語留下的思考題

未來會不會出現「**超級外掛**」——能自己寫程式、跑測試、部署雲端、監控流量、半夜當機自主修復（端到端執行權限）？

屆時人類開發者的核心價值，將從「程式碼編寫者」轉變成什麼樣的 **決策者**？

---

## 一句話總結

本章教你用 `plugin.json` 編排一支「專屬自動化開發團隊」，
但反覆強調 **能力越強、風險越高**——
安全意識（審查碼／驗作者／控權限）才是駕馭這套引擎的前提。
